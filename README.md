# PharmDash

A Next.js 16 pharmaceutical intelligence dashboard for monitoring product listings, domain activity, and social media mentions.

## Getting Started

```bash
cp .env.example .env.local   # set DASHBOARD_LOGIN_USERNAME / DASHBOARD_LOGIN_PASSWORD
npm install
npm run dev
```

Open [http://localhost:3000](http://localhost:3000). Default credentials: `admin / 123456` (override via env).

### Core entity — `Listing`

The finest-granularity unit. Generated by `generateListings()` using a seeded RNG for reproducibility.

```ts
interface Listing {
  id: string;              // "listing-1", "listing-2", …
  detectedAt: Date;        // random day within the month (server-side only, not sent to client)
  source: "online" | "social";
  primaryCategory: string; // "GLP-1" | "Cancer Med" | "CNS Med" | "Pain Med"
  secondaryCategory: string; // e.g. "Ozempic", "Olaparib", "Aripiprazole"
  quarter: string;         // "Q2-2024"
  month: string;           // "April-2024"  (used as chart x-axis key)
}
```

**Current dataset:** 155 listings across April / May / June 2024 (45 / 52 / 58 per month).

#### Primary → Secondary product map

| Primary | Secondary products |
|---|---|
| GLP-1 | Ozempic, Mounjaro, Wegovy, Rybelsus |
| Cancer Med | Olaparib, Pembrolizumab, Nivolumab |
| CNS Med | Aripiprazole, Risperidone, Olanzapine |
| Pain Med | Tramadol, Gabapentin, Pregabalin |

### Derived structures (built from `Listing[]`)

#### `PieChartNodeData` — sunburst tree

```ts
interface PieChartNodeData {
  id: string;         // "GLP-1" (primary) or "GLP-1-Ozempic" (secondary)
  name: string;
  value: number;      // listing count
  percentage: number; // % within parent
  color?: string;     // hex — shared between a primary and its children
  children?: PieChartNodeData[];
}
```

Built by `getDrillablePieData()`. Inner ring = primary category, outer ring = secondary product.

### `SubPageData` — the root record per subpage

```ts
interface SubPageData {
  key: SubPageKey;          // "top-products" | "domain-insights" | "social-media-insights"
  title: string;
  summary: string;
  metrics: MetricCardData[];
  charts: ChartCardData[];
  rankedItems: RankedItem[];
  categories?: CategoryOption[];   // filter dropdown options (top-products only)
  drillablePieData?: PieChartNodeData[]; // sunburst tree (top-products only)
  listings?: Listing[];            // raw data (top-products only, served via API)
}
```

> **Note — top-products:** `metrics`, `charts`, `rankedItems` are intentionally `[]`. The
> `TopProductsSubpage` component derives all three client-side from `listings` via `useMemo`
> so that the category filter works without re-fetching.

## Reporting Period

All period labels (card subtitles, chart titles, sunburst center) read from a single constant:

```ts
// app/dashboard/components/subpages/top-products/config.ts
export const CURRENT_PERIOD = "Q2 2026";
```

Change this one line to update every visible label across all cards.

## Data Flow

```
mock-data.ts  (server module)
  └─ subPageDataMap["top-products"].listings  ──►  GET /api/listings/top-products
                                                        │  verifies pharmdash_auth cookie
                                                        │  strips detectedAt
                                                        ▼
                                               TopProductsSubpage (client)
                                                 useSWR — fetches once on mount
                                                 useMemo — filter / aggregate locally
                                                   ├── MetricsRow
                                                   ├── TopProductsRanked
                                                   ├── ListingTrendChart
                                                   └── ProductDistribution

  └─ subPageDataMap["domain-insights"]   ──►  DomainInsightsSubpage   (direct import)
  └─ subPageDataMap["social-media-..."]  ──►  SocialMediaInsightsSubpage (direct import)
```

## Auth

- Login: `POST /api/auth/login` — validates env credentials, sets `httpOnly` cookie `pharmdash_auth=1`
- Guard: `app/dashboard/page.tsx` (Server Component) reads cookie and `redirect()`s if absent
- API guard: `GET /api/listings/top-products` independently checks the same cookie
